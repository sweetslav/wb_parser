# README.md

# Wildberries → Google Sheets Data Aggregator

Скрипт для автоматического сбора и агрегации данных по рекламным кампаниям Wildberries в Google Таблицах.

## 📋 Описание

Этот скрипт собирает данные из нескольких API Wildberries и объединяет их в единую таблицу для анализа эффективности рекламных кампаний. Основная цель — предоставить маркетологам и владельцам магазинов комплексный отчет по каждому товару, включающий:

- Заказы и выручку
- Затраты на рекламу
- Показатели эффективности (ДРР, CR1)
- Клики и добавления в корзину
- Артикулы продавца (vendorCode)

## 🚀 Возможности

- **Автоматический сбор данных** из 4 API Wildberries:
  - Заказы (statistics-api)
  - Аналитика (seller-analytics-api)
  - Рекламные затраты (advert-api)
  - Карточки товаров (content-api)
- **Умное объединение** данных по артикулам (nmId)
- **Расчет ключевых метрик**:
  - ДРР (Доля рекламных расходов)
  - CR1 (Конверсия из кликов в корзину)
- **Интерактивное меню** в Google Таблицах
- **Автоматическое обновление** данных при нажатии кнопки
- **Логирование** всех этапов работы для отладки

## 📊 Структура таблицы

| Столбец | Название | Описание |
|---------|----------|----------|
| A | Артикул WB (nmId) | ID товара в Wildberries |
| B | Артикул продавца (vendorCode) | Ваш внутренний артикул |
| C | Название | Название товара |
| D | Заказы общие, ₽ | Сумма всех заказов |
| E | Затраты на рекламу, ₽ | Расходы на рекламу |
| F | ДРР | Доля рекламных расходов (%) |
| G | Клики Общие | Количество кликов |
| H | Корзина общая | Добавления в корзину |
| I | CR1 | Конверсия (%) |
| J-M | Доп. поля | Для ручного заполнения |

## 🔧 Установка и настройка

### 1. Создание скрипта в Google Таблицах

1. Откройте Google Таблицу
2. Перейдите в `Расширения → Apps Script`
3. Скопируйте весь код из этого репозитория в редактор
4. Сохраните проект (💾)

### 2. Настройка токена

1. Получите API-токен Wildberries в личном кабинете:
   - Настройки → Доступ к API → Создать токен
   - Скопируйте полученный токен

2. В коде найдите строку:
   ```javascript
   const token = 'ВАШ_КЛЮЧ_WILDBERRIES_СЮДА';
   ```
   Замените `ВАШ_КЛЮЧ_WILDBERRIES_СЮДА` на ваш токен

3. Запустите функцию `setWBToken()`:
   - В редакторе выберите функцию из списка
   - Нажмите `Run`
   - Подтвердите доступ к сервисам Google

4. После сохранения токена **удалите** строку с токеном из кода

### 3. Настройка листа

В Google Таблице создайте лист с названием `adv_effectiveness` и следующей структурой:

| Строка | A | B | C | D | ... |
|--------|---|---|---|-----|
| 1 | | | | | |
| 2 | Дата с: | 2026-01-01 | Дата по: | 2026-01-31 | |
| 3 | | | | | |
| 4 | Артикул WB | Артикул продавца | Название | Заказы общие, ₽ | ... |

**Важно:** Ячейки B2 и C2 должны содержать даты в формате `ГГГГ-ММ-ДД`

## 📖 Использование

### Быстрый старт

1. В Google Таблице перейдите в меню `📊 Мои отчеты`
2. Нажмите `🔄 Обновить данные (v4.0)`
3. Дождитесь завершения загрузки (около 10-30 секунд)
4. Данные автоматически запишутся в таблицу

### Ручной запуск

В редакторе Apps Script:
1. Выберите функцию `main` из списка
2. Нажмите `Run`
3. Следите за логами в `View → Logs`

## 🔍 Особенности работы

### 1. Сдвиг по рекламе (-7 дней)
Рекламные данные запрашиваются со сдвигом на 7 дней назад, так как Wildberries может задерживать обновление статистики по рекламным расходам.

### 2. Распределение рекламных затрат
Если в кампании несколько товаров, затраты распределяются равномерно между ними.

### 3. Загрузка vendorCode
Артикулы продавца загружаются одним запросом для всех товаров, что экономит время и ресурсы API.

## 🛠 Технические детали

### Используемые API

| API | Endpoint | Назначение |
|-----|----------|------------|
| Statistics API | `/api/v1/supplier/orders` | Заказы |
| Analytics API | `/api/analytics/v3/sales-funnel/products` | Клики и корзина |
| Advert API | `/adv/v1/upd` | Рекламные затраты |
| Content API | `/content/v2/get/cards/list` | Артикулы продавца |

### Ограничения

- Максимальное количество записей за один запрос: 1000
- Период аналитики ограничен API Wildberries
- Для больших объемов данных может потребоваться пагинация

## 🐛 Отладка

### Встроенные тесты

В коде есть тестовые функции для проверки отдельных компонентов:

- `testAllCardsVendorCodes()` — проверка получения vendorCode
- `testCampaignsCount()` — проверка количества кампаний
- `testUpd()` — проверка получения рекламных затрат

### Логирование

Все этапы работы логируются. Для просмотра логов:
1. В редакторе Apps Script
2. `View → Logs`
3. Или `View → Executions` для детальной информации
