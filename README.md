## 📖 Описание проекта

Скрипт автоматизирует сбор данных из Wildberries API в Google Sheets. Поддерживает два кабинета (ООО и ИП) на двух разных листах.

### Ключевые возможности:
- ✅ Автоматический сбор данных из **5 API источников**
- ✅ Поддержка **двух кабинетов** на разных листах
- ✅ **26 финансовых показателей** по каждому товару
- ✅ **Прогресс-бар** в таблице для отслеживания статуса
- ✅ **Защищенное хранение токенов** через `PropertiesService`
- ✅ Все числа **целые** (без десятичных знаков)

| Z | Маржинальность, % | Расчет | - |

### Формулы расчетов

```javascript
// ДРР
ДРР = Затраты на рекламу / Заказы * 100

// CR1
CR1 = Корзина / Клики * 100

// Цена полки
Цена полки = Продажи после СПП / Количество продаж

// Валовая маржа
Валовая маржа = Продажи после СПП 
  - Себестоимость 
  - Логистика 
  - Комиссия 
  - Штрафы 
  - Хранение 
  - Удержания 
  - Приемка 
  - Реклама 
  + Компенсации

// Маржинальность
Маржинальность = Валовая маржа / Продажи после СПП * 100
```

## 🔧 Установка и настройка

### 1. Создание таблицы

1. Открой **Google Sheets**
2. Создай новую таблицу
3. Создай листы:
   - `adv_effectiveness` (кабинет 1)
   - `adv_effectiveness_2` (кабинет 2)
4. Настрой заголовки (см. структуру выше)

### 2. Настройка справочника

Создай отдельную таблицу с ID: `упс это конфиденциальная информация`

Лист: `unit расчет` со столбцами:
- **M (12)** = nmId (артикул WB)
- **Y (24)** = себест100 (себестоимость за 1 шт)
- **Z (25)** = FBW (остаток на складах WB)
- **AC (28)** = FBS (остаток 1С)

### 3. Получение токенов Wildberries

1. Войди в [личный кабинет Wildberries](https://seller.wildberries.ru)
2. Перейди в **Настройки → Доступ к API**
3. Создай токен с правами:
   - Статистика
   - Аналитика
   - Финансы
   - Документы

**Важно:** Для двух кабинетов нужны два разных токена.

### 4. Установка скрипта

1. В Google Sheets: **Расширения → Apps Script**
2. Скопируй весь код из [`src/main.gs`](src/main.gs)
3. Сохрани (Ctrl+S)
4. Обнови таблицу (F5)

## 📱 Использование

### Меню "📊 Мои отчеты"

После установки появится меню с пунктами:

| Пункт | Действие |
|-------|----------|
| **Кабинет 1"** | Обновить кабинет 1 |
| **Кабинет 2"** | Обновить кабинет 2 |
| **Обновить все** | Обновить оба кабинета |
| **Токен кабинет 1** | Установить токен для кабинета 1 |
| **Токен кабинет 2** | Установить токен для кабинета 2 |
| **Очистить статусы** | Очистить ячейки A3-C3 |

### Процесс обновления

1. Установи даты в B2 и C2 (период не более 31 дня)
2. Нажми **"Обновить все"**
3. Следи за прогрессом в строке 3:
   - **A3**: Текущий статус
   - **B3**: Прогресс-бар
   - **C3**: Текущий этап

### Этапы обновления

| # | Этап | Описание |
|---|------|----------|
| 1 | Заказы | API заказов (названия товаров) |
| 2 | Аналитика | API аналитики (клики, корзина, заказы) |
| 3 | Реклама | API рекламы (расходы по кампаниям) |
| 4 | Артикулы | API цен (vendorCode) |
| 5 | Остатки | Справочник (остатки + себестоимость) |
| 6 | Финансы | API финансов (детализация отчетов) |
| 7 | Объединение | Склейка данных и запись в таблицу |

```

│   └── runForCabinet()           # Универсальная функция для любого кабинета

│       │   └── aggregateFinancialData()

├── onOpen()                      # Создание меню
└── test*()                       # Тестовые функции
```

## 🌐 API Источники

| Источник | URL | Назначение |
|----------|-----|------------|
| Заказы | `statistics-api.wildberries.ru/api/v1/supplier/orders` | Названия товаров |
| Аналитика | `seller-analytics-api.wildberries.ru/api/analytics/v3/sales-funnel/products` | Клики, корзина, заказы |
| Реклама | `advert-api.wildberries.ru/adv/v1/upd` | Расходы на рекламу |
| Кампании | `advert-api.wildberries.ru/api/advert/v2/adverts` | Состав кампаний |
| Цены | `discounts-prices-api.wildberries.ru/api/v2/list/goods/filter` | vendorCode |
| Финансы | `finance-api.wildberries.ru/api/finance/v1/sales-reports/detailed` | Детализация отчетов |

| Поле | Описание | Наш столбец |

| `ppvzSalesCommission` | Вознаграждение с продаж | R: Комиссия |

| `paidStorage` | Хранение | T: Хранение |
| `deduction` | Удержания | U: Удержания |
| `paidAcceptance` | Операции на приёмке | V: Приемка |
| `additionalPayment` | Корректировка ВВ | X: Компенсации |

## ⚡ Лимиты и ограничения

### API Лимиты Wildberries

| API | Персональный токен | Базовый токен |
|-----|-------------------|---------------|
| Заказы | 1 запрос/мин | 1 запрос/3 часа |
| Аналитика | 1 запрос/мин | 1 запрос/24 часа |
| Реклама | 1 запрос/мин | 1 запрос/24 часа |
| Финансы | 1 запрос/мин | 1 запрос/24 часа |
| Цены | 1 запрос/мин | 1 запрос/24 часа |

### Ограничения скрипта

- **Максимальный период:** 31 день (защита от перегрузки)
- **Максимальное количество товаров:** зависит от API (пагинация 1000 записей)
- **Время выполнения:** до 10 минут на один кабинет

### Задержки между запросами

```javascript
const DELAYS = {
```

```javascript
function setWBToken3() {
```

```javascript
menu.addItem('🔄 Новый кабинет', 'mainCab3');
```

ID справочника задан в `fetchStockData()`:
```javascript
const SPREADSHEET_ID = 'ВАШ ID';
```

Замени на ID своей таблицы.

## 📝 Примечания

1. **СПП** — Скидка постоянного покупателя (скидки WB)
2. **Реализация до СПП** — сумма в розничных ценах (retailPrice)
3. **Продажи после СПП** — фактическая цена покупателя (retailAmount)
4. **Реклама** не входит в "Итого к оплате" в отчетах WB
5. **Все числа** округлены до целых для читаемости
